ADT vs. C++ class 完整說明

ADT（Abstract Data Type，抽象資料型態）強調的是「這個資料可以做什麼」，而不是「它在記憶體裡到底怎麼存」。也就是說，ADT重視的是資料物件本身的意義、可執行的操作、輸入輸出規格與行為規則。例如投影片中的 NaturalNumber ADT，會先定義物件是從 0 到 MAXINT 的整數範圍，再定義 Zero()、IsZero()、Add()、Equal()、Successor()、Subtract() 等操作，這些都是從「規格」角度描述資料型態，而不是先管它內部怎麼存。

相對地，C++ class 是用來實作這個 ADT 的工具。投影片指出，一個 C++ class 通常包含四個核心部分：類別名稱、資料成員、成員函數，以及存取權限控制（public / private / protected）。其中資料成員構成物件的內部狀態，成員函數則提供外部可呼叫的操作，而 public、private、protected 用來控制外部程式能不能直接接觸內部資料。

兩者最大的差別在於層次不同。ADT 屬於概念層、設計層，它只告訴我們有哪些合法操作、每個操作要達成什麼效果；C++ class 則屬於語言層、實作層，它把 ADT 真正寫成程式。投影片也特別提到資料抽象（data abstraction）與資料封裝（data encapsulation）：資料抽象是把操作規格和實作分開，規格放在類別定義的 public 部分，並描述函式做什麼，但不說內部細節；資料封裝則是把資料成員設成 private 或 protected，再透過 public 成員函數讓外部使用。

所以，ADT 是「規則與介面」；C++ class 是「用 C++ 把規則做出來」。可以把 ADT 想成藍圖、需求書；把 C++ class 想成照藍圖蓋出來的房子。藍圖會說房子要有客廳、房間、浴室，但不一定先講管線怎麼走；真正施工時才決定牆壁厚度、鋼筋配置、材料等。同理，ADT 只要求有哪些操作；C++ class 才決定成員變數如何存、函式如何寫、是否使用建構子、解構子、運算子多載等。投影片中的 Rectangle 類別就是例子：它有資料成員 xLow, yLow, height, width，有建構子、解構子、GetHeight()、GetWidth()，還可用 . 與 -> 存取物件成員，這些都是 class 的具體實作形式。

結論

總結來說，ADT 與 C++ class 不是互相對立，而是上下層的關係。ADT 負責定義資料型態的行為與功能，是抽象的規格；C++ class 負責把這份規格落實成真正可執行的程式，是具體的實作。寫程式時，若先從 ADT 思考，就能清楚知道這個資料型態應提供哪些操作；再用 C++ class 去封裝資料、設計介面、隱藏內部細節，程式就會更模組化、更安全，也更容易維護。因此可以說：ADT 是設計思想，C++ class 是實現方法；class 是實作 ADT 最常見、最自然的工具。

#include <iostream>
using namespace std;

class NaturalNumber {
private:
    // 資料成員：真正儲存數值的地方
    // 設成 private，外部不能直接亂改，這就是封裝
    unsigned int value;

    // 設定最大值，模擬 ADT 中的 MAXINT 概念
    static const unsigned int MAX_VALUE = 100;

public:
    // 建構子：建立物件時初始化 value
    // 若沒有給值，預設為 0
    NaturalNumber(unsigned int v = 0) {
        if (v <= MAX_VALUE)
            value = v;
        else
            value = MAX_VALUE;  // 超過上限就設成最大值
    }

    // Zero(): 回傳一個值為 0 的 NaturalNumber
    static NaturalNumber Zero() {
        return NaturalNumber(0);
    }

    // IsZero(): 判斷目前物件是否為 0
    bool IsZero() const {
        return value == 0;
    }

    // Add(y): 回傳 *this + y，若超過 MAX_VALUE 就回傳 MAX_VALUE
    NaturalNumber Add(const NaturalNumber& y) const {
        unsigned int sum = value + y.value;
        if (sum > MAX_VALUE)
            sum = MAX_VALUE;
        return NaturalNumber(sum);
    }

    // Equal(y): 判斷兩個 NaturalNumber 是否相等
    bool Equal(const NaturalNumber& y) const {
        return value == y.value;
    }

    // Successor(): 若已達最大值就維持不變，否則加 1
    NaturalNumber Successor() const {
        if (value == MAX_VALUE)
            return NaturalNumber(MAX_VALUE);
        return NaturalNumber(value + 1);
    }

    // Subtract(y): 若 *this < y，回傳 0；否則回傳差值
    NaturalNumber Subtract(const NaturalNumber& y) const {
        if (value < y.value)
            return NaturalNumber(0);
        return NaturalNumber(value - y.value);
    }

    // 額外提供一個顯示函式，方便輸出結果
    void Print() const {
        cout << value;
    }
};

int main() {
    // 建立兩個自然數物件
    NaturalNumber a(10);
    NaturalNumber b(25);

    // 測試 Add
    NaturalNumber c = a.Add(b);
    cout << "a + b = ";
    c.Print();
    cout << endl;

    // 測試 Successor
    NaturalNumber d = a.Successor();
    cout << "successor of a = ";
    d.Print();
    cout << endl;

    // 測試 Subtract
    NaturalNumber e = a.Subtract(b);
    cout << "a - b = ";
    e.Print();
    cout << endl;

    // 測試 IsZero
    NaturalNumber z = NaturalNumber::Zero();
    cout << "z = ";
    z.Print();
    cout << endl;
    cout << "Is z zero? " << (z.IsZero() ? "true" : "false") << endl;

    // 測試 Equal
    cout << "Is a equal to b? " << (a.Equal(b) ? "true" : "false") << endl;

    return 0;
}

C++ class 則是用來實現 ADT 的工具，屬於具體的程式設計機制。類別包含資料成員（data members）與成員函數（member functions），並透過 public / private / protected 控制存取權限，使資料受到保護並透過函式操作。

ADT 屬於抽象層（設計層），只定義資料與操作的規格；
C++ class 屬於實作層（程式層），負責將這些規格轉換為實際程式碼。

  1. 資料封裝（Encapsulation）：將資料成員設為 private，避免外部直接存取

因此，ADT 與 C++ class 的關係可總結為：
👉 ADT 是規格（Specification），C++ class 是實作（Implementation）

🧠 Data Abstraction 與 Encapsulation 完整說明

在物件導向程式設計中，**Data Abstraction（資料抽象）與Encapsulation（資料封裝）**是兩個核心概念，兩者相互配合，用來提升程式的可維護性與安全性。

**資料抽象（Data Abstraction）**指的是：
👉 只描述「功能與操作」，不揭露內部實作細節

也就是說，使用者只需要知道「這個物件可以做什麼」，而不需要知道「它是怎麼做的」。例如在 ADT 中，只會定義操作名稱、參數與回傳型態，而不說明內部如何儲存資料或計算結果。投影片也指出，操作規格應寫在 class 的 public 部分，並用註解說明功能，但不涉及實作細節。

**資料封裝（Encapsulation）**則是：
👉 把資料與操作包在一起，並隱藏內部資料

通常會將資料成員設為 private，外部程式無法直接存取，必須透過 public 成員函數操作。這樣可以避免資料被任意修改，提升安全性。投影片也強調，資料封裝是透過將資料設為 private，並提供函式作為唯一存取管道來實現。

  • Abstraction：決定要提供哪些功能（介面）

  • Encapsulation：保護內部資料不被亂用

👉 抽象是「對外呈現」，封裝是「對內保護」

🧠 Array as ADT 完整說明

在資料結構中，陣列（Array）不只是「一段連續記憶體」，而是可以從抽象資料型態（ADT）的角度來理解。若以 ADT 觀點來看，陣列是一個由索引（index）對應到值（value）的映射（mapping），也就是一組 <index, value> 的配對集合。每一個合法的 index，都會對應到一個值，而這種對應關係才是陣列的本質，而非其記憶體配置方式。

在 ADT 的觀點中，重點在於「可以對陣列做哪些操作」，而不是「資料如何存」。常見的操作包含：

  1. Retrieve（取值）：根據 index 取得對應的值

  2. Store（存值）：將某個值存入指定 index

  3. （延伸）Insert、Delete、Traverse 等

這些操作定義了陣列的行為，而內部是否用連續記憶體、是否動態配置，都是實作細節，不屬於 ADT 範疇。投影片也指出，多數語言對陣列只提供 Retrieve 與 Store 兩種基本操作，這正符合 ADT 的設計精神。

⚖️ Array（實作） vs Array as ADT（抽象）

項目一般陣列Array as ADT
重點記憶體配置功能與操作
表示方式連續記憶體<index, value>
限制index 必須連續可自訂 index 集合
安全性不檢查越界可設計例外處理

👉 例如：C++ 陣列 a[i] 不會檢查 index 是否超出範圍，但 ADT 可以設計成「若 index 不合法就拋出錯誤」，因此更安全、更通用。

🎯 結論（申論收尾必寫🔥）

將陣列視為 ADT，可以將重點從「資料如何儲存」轉移到「資料如何操作」，使設計更具彈性與抽象性。ADT 的概念讓陣列不再只是固定結構，而是一種可擴展的資料模型，能支援多維、非連續索引與錯誤檢查。因此，Array as ADT 提供更高層次的設計思維，使程式更安全、模組化且易於維護。

C++
#include <iostream>
#include <vector>
using namespace std;

// Array ADT 類別
class GeneralArray {
private:
    vector<float> data;   // 實際儲存資料
    int size;             // 陣列大小

public:
    // 建構子：建立大小為 n 的陣列，初始值為 initValue
    GeneralArray(int n, float initValue = 0.0) {
        size = n;
        data.resize(n, initValue);  // 初始化每個元素
    }

    // Retrieve：取得第 i 個元素
    float Retrieve(int i) {
        if (i < 0 || i >= size) {
            cout << "錯誤：index 超出範圍！" << endl;
            exit(1); // 結束程式（也可以改成 exception）
        }
        return data[i];
    }

    // Store：將值存入第 i 個位置
    void Store(int i, float value) {
        if (i < 0 || i >= size) {
            cout << "錯誤：index 超出範圍！" << endl;
            exit(1);
        }
        data[i] = value;
    }

    // 額外功能：印出整個陣列
    void Print() {
        for (int i = 0; i < size; i++) {
            cout << data[i] << " ";
        }
        cout << endl;
    }
};

int main() {
    // 建立一個大小為 5 的 Array ADT
    GeneralArray arr(5, 1.0); // 初始值全部為 1.0

    // 存值
    arr.Store(2, 10.5);
    arr.Store(4, 20.8);

    // 取值
    cout << "arr[2] = " << arr.Retrieve(2) << endl;
    cout << "arr[4] = " << arr.Retrieve(4) << endl;

    // 印出整個陣列
    arr.Print();

    return 0;
}

陣列（Array）除了基本儲存資料外，在資料結構中常被用來實作多種抽象資料型態（ADT），其中最重要的應用包括有序串列（Ordered Lists）、多項式（Polynomials）與稀疏矩陣（Sparse Matrices）。這些應用都利用陣列「快速存取（O(1)）」的特性，但也各自面臨不同設計考量。

📌 1. Ordered Lists（有序串列）

有序串列是一組依序排列的資料，例如 (a0, a1, a2, …, an)，可用陣列直接對應 index 與元素。常見操作包括存取、插入、刪除等。使用陣列實作時：

  • 存取（Retrieve）：O(1)

  • 插入 / 刪除：O(n)，因為需搬移元素

👉 優點：存取速度快
👉 缺點：修改成本高（shift elements）

📌 2. Polynomials（多項式）

多項式可表示為：

a(x)=a0+a1x+a2x2+⋯a(x) = a_0 + a_1x + a_2x^2 + \cdotsa(x)=a0​+a1​x+a2​x2+⋯

可用陣列儲存係數，例如 coef[i] 代表 xix^ixi 的係數。

但會遇到兩種情況：

  • Dense（稠密）：大多數項非 0 → 用陣列有效

  • Sparse（稀疏）：大多數項為 0 → 浪費空間

因此改用「(coef, exp) 陣列」只存非零項，更有效率。

👉 多項式加法可用「merge」方式，時間複雜度 O(m+n)

📌 3. Sparse Matrices（稀疏矩陣）

當矩陣中大部分元素為 0（例如 5000×5000 但只有 5000 個非零），若使用一般 2D array 會嚴重浪費空間。

因此使用「三元組表示法」：

(row, col, value)

👉 優點：

  • 節省記憶體

  • 操作（如轉置、加法）更有效率

例如轉置矩陣可透過重新排列三元組完成。

🎯 結論（申論必寫🔥）

陣列不僅是基本資料結構，更是實作多種 ADT 的核心工具。在有序串列中提供快速存取，在多項式中支援數學運算，在稀疏矩陣中透過壓縮表示提升效率。不同應用會依資料特性選擇適當表示方式，例如 dense 使用一般陣列，sparse 使用壓縮結構。整體而言，陣列的應用關鍵在於平衡時間效率與空間使用，使其在各種問題中達到最佳效能。

#include <iostream>
#include <vector>
using namespace std;

// 表示一個多項式項目 (coef, exp)
struct Term {
    int coef; // 係數
    int exp;  // 指數
};

class Polynomial {
private:
    vector<Term> terms; // 儲存所有非零項

public:
    // 新增一個項目
    void addTerm(int coef, int exp) {
        if (coef != 0) {
            terms.push_back({coef, exp});
        }
    }

    // 多項式加法（使用 merge 概念）
    Polynomial add(Polynomial& b) {
        Polynomial result;
        int i = 0, j = 0;

        while (i < terms.size() && j < b.terms.size()) {
            if (terms[i].exp == b.terms[j].exp) {
                int sum = terms[i].coef + b.terms[j].coef;
                if (sum != 0)
                    result.addTerm(sum, terms[i].exp);
                i++; j++;
            }
            else if (terms[i].exp > b.terms[j].exp) {
                result.addTerm(terms[i].coef, terms[i].exp);
                i++;
            }
            else {
                result.addTerm(b.terms[j].coef, b.terms[j].exp);
                j++;
            }
        }

        // 加入剩餘項
        while (i < terms.size()) {
            result.addTerm(terms[i].coef, terms[i].exp);
            i++;
        }
        while (j < b.terms.size()) {
            result.addTerm(b.terms[j].coef, b.terms[j].exp);
            j++;
        }

        return result;
    }

    // 印出多項式
    void print() {
        for (auto t : terms) {
            cout << t.coef << "x^" << t.exp << " ";
        }
        cout << endl;
    }
};

int main() {
    Polynomial a, b;

    // a(x) = 3x^2 + 2x
    a.addTerm(3, 2);
    a.addTerm(2, 1);

    // b(x) = x^2 + 5
    b.addTerm(1, 2);
    b.addTerm(5, 0);

    Polynomial c = a.add(b);

    cout << "a(x) + b(x) = ";
    c.print();

    return 0;
}

💻 完整程式（C++＋中文註解）

C++
#include <iostream>
#include <stack>
#include <queue>
using namespace std;

// ADT: TaskSystem
class TaskSystem {
private:
    stack<string> urgentStack; // 緊急任務（LIFO）
    queue<string> normalQueue; // 一般任務（FIFO）

public:
    // 新增任務
    void AddTask(string task, bool isUrgent) {
        if (isUrgent) {
            urgentStack.push(task); // 放入 Stack
        } else {
            normalQueue.push(task); // 放入 Queue
        }
    }

    // 處理任務
    void ProcessTask() {
        if (!urgentStack.empty()) {
            // 優先處理緊急任務（Stack）
            cout << "處理緊急任務: " << urgentStack.top() << endl;
            urgentStack.pop();
        }
        else if (!normalQueue.empty()) {
            // 若無緊急任務，處理一般任務（Queue）
            cout << "處理一般任務: " << normalQueue.front() << endl;
            normalQueue.pop();
        }
        else {
            cout << "沒有任務可處理" << endl;
        }
    }

    // 判斷是否為空
    bool IsEmpty() {
        return urgentStack.empty() && normalQueue.empty();
    }
};

int main() {
    TaskSystem system;

    // 新增任務
    system.AddTask("寫作業", false);
    system.AddTask("打電動", false);
    system.AddTask("交報告", true); // 緊急
    system.AddTask("考試", true);   // 緊急

    // 處理任務
    while (!system.IsEmpty()) {
        system.ProcessTask();
    }

    return 0;
}

Array representation 完整說明

在資料結構中，Array representation（陣列表示法）主要是在討論：一個一維、二維，甚至多維陣列，如何在電腦記憶體中安排與定位。雖然我們在程式裡看到的是 a[i]、a[i][j]、a[i][j][k] 這種形式，但實際上電腦記憶體仍然是線性的一維空間，因此多維陣列最後都必須轉換成一維位置來儲存。投影片指出，二維陣列可依照 row major（列優先／行主序） 或 column major（行優先／列主序） 來排列；而多維陣列也可透過同樣概念，將 n 維陣列映射到一維記憶體。

1. 二維陣列的表示

以投影片中的例子來看，若宣告 a[2:6, 3:5]，表示 row 範圍是 2 到 6，col 範圍是 3 到 5，因此元素總數為：

(6−2+1)×(5−3+1)=5×3=15(6-2+1)\times(5-3+1)=5\times3=15(6−2+1)×(5−3+1)=5×3=15

也就是共有 15 個元素。

(1) Row major

在 row major 中，記憶體依照「先列後行」方式儲存，也就是先把同一列的所有元素依序放完，再放下一列。例如：

a[2][3], a[2][4], a[2][5], a[3][3], a[3][4], ...

這也是 C/C++ 常用的排列方式。

(2) Column major

在 column major 中，記憶體依照「先行後列」方式儲存，也就是先把同一行的元素依序放完，再放下一行。例如：

a[2][3], a[3][3], ..., a[6][3], a[2][4], ...

這種方式在某些語言或數學軟體中較常見。

2. 位址計算公式

投影片也給出二維與三維陣列的位址公式。若一個二維陣列宣告為 a[u1][u2]，起始位址為 α，則在 row major 下：

Address(a[i][j])=α+i⋅u2+jAddress(a[i][j]) = \alpha + i \cdot u2 + jAddress(a[i][j])=α+i⋅u2+j

若是三維陣列 a[u1][u2][u3]，則：

Address(a[i][j][k])=α+i⋅u2⋅u3+j⋅u3+kAddress(a[i][j][k]) = \alpha + i \cdot u2 \cdot u3 + j \cdot u3 + kAddress(a[i][j][k])=α+i⋅u2⋅u3+j⋅u3+k

這表示二維以上的陣列，本質上都是把前面維度的元素數量算進偏移量，再加上最後一維的位置。

3. 多維陣列的表示

對於一般 n 維陣列 a[u1][u2]...[un]，在 row major 下，其位址可表示成：

α+i1u2u3...un+i2u3u4...un+...+in−1un+in\alpha + i_1u_2u_3...u_n + i_2u_3u_4...u_n + ... + i_{n-1}u_n + i_nα+i1​u2​u3​...un​+i2​u3​u4​...un​+...+in−1​un​+in​

這個公式的重點是：越前面的索引，影響的跨度越大；越後面的索引，影響的跨度越小。
也就是說，第一維索引決定你跳過多少個大區塊，最後一維索引則只是區塊內的小位移。

因為它不只是數學公式，而是很多資料結構與演算法的基礎。例如：

簡單說，Array representation 是把抽象的多維結構，轉成具體記憶體位址的方法。

結論

總結來說，Array representation 的核心在於：多維陣列雖然在邏輯上看起來像表格或立方體，但在實際記憶體中仍是線性排列。 因此，必須透過 row major 或 column major 將索引轉成位址。C/C++ 常用 row major，所以二維陣列會按列依序存放，三維以上則依公式層層展開。理解這個概念後，不但能掌握陣列本質，也能更清楚知道多維資料在電腦內部的真正表示方式。

Strings 與 String Pattern Matching 完整說明

在資料結構中，String（字串）可視為一種特殊的線性資料結構，它是由零個或多個字元依照順序排列而成的集合。投影片中將 ADT String 定義為：若字串為 S = s0, s1, ..., sn-1，則當 n = 0 時稱為空字串（empty/null string）。同時，String ADT 提供多種基本操作，例如建構字串、判斷是否相等、判斷是否為空、求長度、串接、取子字串，以及尋找某個 pattern 是否出現在原字串中。這表示字串不只是字元的集合，更是一種具有明確操作規格的抽象資料型態。

在這些操作中，**String pattern matching（字串樣式比對）**是非常重要的一項。它的目的就是：在一個主字串（text/string）中，找出某個模式字串（pattern）是否出現，以及出現的位置。例如在字串 "abaabbaabaa" 中尋找 "aab"，系統就必須從某個起始位置開始逐字比較，判斷 pattern 是否完整匹配。投影片指出，最直接的方法是 Exhaustive Pattern Matching（暴力比對法），其想法很簡單：從主字串每一個可能的起始位置開始，一個字一個字與 pattern 比較；若中途不相等，就換下一個起點重新開始。這種方法雖然容易理解與實作，但時間複雜度是：

O(lengthP×lengthS)O(lengthP \times lengthS)O(lengthP×lengthS)

其中 lengthP 是 pattern 長度，lengthS 是主字串長度。當字串很長時，效率可能較差。

為了改進效率，投影片接著介紹 Knuth-Morris-Pratt Algorithm（KMP 演算法）。KMP 的核心概念是：當比對失敗時，不要把已經知道的資訊全部丟掉，而是利用 pattern 自己的結構，決定下一步從哪裡繼續比。 為了做到這件事，KMP 會先對 pattern 做前處理，建立一個 failure function（失敗函數）。這個函數記錄：當 pattern 比對到某位置失敗時，可以退回到哪個位置重新繼續，而不用從頭開始。投影片中以 pattern abcabcacab 為例，列出每個位置對應的 failure value，並說明若目前已比對一部分成功，但下一個字元失敗，就可依 failure function 決定新的 pattern 位置，而主字串指標不必大幅回退。

KMP 演算法分成兩部分：
第一部分是 fail()，用來預先計算 pattern 的 failure function，其時間複雜度為：

O(lengthP)O(lengthP)O(lengthP)

第二部分是 FastFind()，真正執行匹配，其時間複雜度為：

O(lengthS)O(lengthS)O(lengthS)

因此整體 KMP 的時間複雜度為：

O(lengthP+lengthS)O(lengthP + lengthS)O(lengthP+lengthS)

這比暴力法的 O(lengthP × lengthS) 更有效率，尤其當 pattern 與主字串都很長時，優勢非常明顯。

結論

總結來說，String 是由有序字元組成的抽象資料型態，除了基本操作如長度、串接與子字串外，最重要的應用之一就是 pattern matching。暴力比對法概念直觀、實作簡單，但效率較低；KMP 則利用 failure function 避免重複比較，大幅提升搜尋效率。因此，字串模式比對的核心重點在於如何利用已知匹配資訊減少不必要的重新比較，而 KMP 正是這種思想的代表演算法。 在考試與實務中，若要求效率分析，通常都要特別指出：暴力法是 O(mn)，KMP 是 O(m+n)。

#include <iostream>
#include <vector>
#include <string>
using namespace std;

// ==============================
// 暴力法：Exhaustive Pattern Matching
// 時間複雜度 O(lengthP * lengthS)
// ==============================
int naiveFind(const string& text, const string& pattern) {
    // 若 pattern 為空，直接回傳 -1（也可依需求改成 0）
    if (pattern.empty()) return -1;

    int n = text.length();
    int m = pattern.length();

    // 從 text 的每個可能起點開始比對
    for (int start = 0; start <= n - m; start++) {
        int j = 0;

        // 一個字一個字比對
        while (j < m && text[start + j] == pattern[j]) {
            j++;
        }

        // 若 j == m，表示整個 pattern 都匹配成功
        if (j == m) {
            return start;
        }
    }

    // 找不到
    return -1;
}

// ==============================
// 建立 KMP 的 failure function
// failure[i] 表示 pattern[0..i] 中
// 最長相同前綴與後綴的位置資訊
// 時間複雜度 O(lengthP)
// ==============================
vector<int> buildFailure(const string& pattern) {
    int m = pattern.length();
    vector<int> failure(m, -1);

    // 從第 1 個位置開始計算
    for (int j = 1; j < m; j++) {
        int i = failure[j - 1];

        // 一直往前找可接續的位置
        while (i >= 0 && pattern[j] != pattern[i + 1]) {
            i = failure[i];
        }

        // 若找到可延續的前綴/後綴
        if (pattern[j] == pattern[i + 1]) {
            failure[j] = i + 1;
        } else {
            failure[j] = -1;
        }
    }

    return failure;
}

// ==============================
// KMP 搜尋
// 整體時間複雜度 O(lengthP + lengthS)
// ==============================
int kmpFind(const string& text, const string& pattern) {
    if (pattern.empty()) return -1;

    int n = text.length();
    int m = pattern.length();

    // 先建立 failure function
    vector<int> failure = buildFailure(pattern);

    int posText = 0;    // text 目前比對位置
    int posPattern = 0; // pattern 目前比對位置

    while (posText < n) {
        // 若字元相同，兩邊一起前進
        if (text[posText] == pattern[posPattern]) {
            posText++;
            posPattern++;

            // 若 pattern 全部匹配完成
            if (posPattern == m) {
                return posText - m;
            }
        }
        else {
            // 不相同時
            if (posPattern == 0) {
                // pattern 一開始就失敗，text 往後一格
                posText++;
            } else {
                // 利用 failure function 跳轉
                posPattern = failure[posPattern - 1] + 1;
            }
        }
    }

    return -1; // 找不到
}

int main() {
    string text = "abaabbaabaa";
    string pattern = "aab";

    cout << "主字串 text = " << text << endl;
    cout << "樣式字串 pattern = " << pattern << endl;

    // 暴力法
    int pos1 = naiveFind(text, pattern);
    if (pos1 != -1)
        cout << "[暴力法] 找到位置: " << pos1 << endl;
    else
        cout << "[暴力法] 找不到 pattern" << endl;

    // 印出 failure function
    vector<int> failure = buildFailure(pattern);
    cout << "KMP failure function: ";
    for (int x : failure) {
        cout << x << " ";
    }
    cout << endl;

    // KMP
    int pos2 = kmpFind(text, pattern);
    if (pos2 != -1)
        cout << "[KMP] 找到位置: " << pos2 << endl;
    else
        cout << "[KMP] 找不到 pattern" << endl;

    return 0;
}

這份程式先用 naiveFind() 實作暴力法。它的做法就是從主字串的每個可能位置開始，把 pattern 從頭拿來比，因此很直觀，但在最差情況下會有大量重複比較。
